Reusable Navigation Tabs (HTML, CSS, and TypeScript)

Overview

The Reusable Navigation Tabs component provides a fully customizable and lightweight tab navigation system built with pure HTML, CSS, and Angular logic. It supports dynamic icons, tab disabling, show/hide control, and event emission when a tab is selected — all without using any third-party libraries.

HTML Template

This structure defines the overall layout of the navigation tabs.

  • The container .nav-tabs-wrapper holds the tab list.
  • Each tab is created dynamically using *ngFor.
  • Tabs can be conditionally shown or disabled using the show and disabled properties.
  • The selectTab() method is triggered when a tab is clicked.

Structure Description:

  • <ul class="nav-tabs"> — Wrapper for all tabs.
  • <li class="nav-tab"> — Individual tab item.
  • <button class="nav-tab-link"> — Clickable tab button that can show an icon and label.
  • <i> — Displays an icon for the tab.
  • <span> — Displays the tab label text.

CSS Styles

The styling defines a clean and modern tab design with hover effects, disabled states, and smooth animations.

Key Style Features:

  • Flex layout for horizontal tab alignment.
  • Border-bottom highlights the active tab visually.
  • Hover effects improve interactivity (background: #f9f9f9).
  • Smooth transitions using transition: all 0.3s ease.
  • Disabled tabs appear dimmed and non-interactive.
  • Fade-in animation for tab content switching.

Class Responsibilities:

  • .nav-tabs-wrapper: Container width and alignment.
  • .nav-tabs: Flexbox container for tab list.
  • .nav-tab-link.active: Highlights the selected tab.
  • .nav-tab-link:disabled: Prevents interaction on disabled tabs.
  • .tab-content.active: Displays active tab content with animation.

TypeScript Logic

This section handles the tab behavior and event management.

Inputs:

  • tabs: Array of tab configuration objects (each with label, icon, show, disabled).
  • activeTab: Index of the currently active tab.

Output:

  • onSelect: Emits an event when a new tab is selected, returning the selected index and tab object.

Logic Flow:

  1. The selectTab() method checks if the clicked tab is disabled.
  2. If enabled, it updates activeTab with the clicked tab index.
  3. It emits the selection event with the tab’s data.
  4. The parent component can listen to onSelect to handle custom logic (e.g., loading different content).

Example Usage

You can integrate this component in your parent template like below: 

<app-nav-tabs
  [tabs]="tabItems"
  [activeTab]="0"
  (onSelect)="handleTabSelection($event)">
</app-nav-tabs>
