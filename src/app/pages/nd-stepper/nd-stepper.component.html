Stepper Component Overview

The Dynamic Stepper Component provides a clean and customizable step-by-step navigation experience for multi-stage workflows such as forms, data wizards, or configurations. Steps can be added dynamically through an input array and can contain icons, labels, and interactive forms or content for each stage.

HTML Template

The component layout uses a simple flex-based design for easy responsiveness. Each step includes an icon, label, and connector line to show progress. The stepper allows users to navigate forward or backward with arrow buttons or by clicking any step.

Key Layout Features:

  • Fully dynamic step generation from an input array.
  • Active and completed step highlighting.
  • Forward/backward navigation using arrow buttons.
  • Click-based direct navigation between steps.

CSS Styles

The stepper uses a centered horizontal layout with hover transitions and color-coded states for completed and active steps.

  • Green for active steps.
  • Orange for completed steps.
  • Gray for pending steps.

TypeScript Logic

The component receives an array of steps as input and emits the current step index through an output event. Parent components can use this to load dynamic content for each step.

Stepper Configuration Options

Property Type Default Description
steps Step[] [] Array of steps containing label and icon for each step.
currentStepIndex number 0 Indicates the current active step index.

Stepper Output Events

Event Type Description
stepChange EventEmitter<number> Emits the current active step index whenever navigation occurs.

Example Usage

Below is an example showing how to integrate the stepper component dynamically with a parent component. 

<app-stepper
  [steps]="steps"
  [currentStepIndex]="activeStep"
  (stepChange)="onStepChange($event)">
</app-stepper>
